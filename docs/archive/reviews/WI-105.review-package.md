## 3. Edge Cases & Design Decisions

- **BigInt Support**: Used `BigInt` for all address calculations to support 64-bit memory spaces without precision loss.
- **Virtual Scroll Strategy**: Used standard `cdk-virtual-scroll-viewport` with `itemSize` fixed at 24px (synced with SCSS). This ensures $O(1)$ DOM complexity regardless of buffer size.
- **Row Padding**: Implemented automatic padding for the final row if the buffer size is not a multiple of 16, ensuring the UI grid remains aligned.
- 🔍 **Inspect**: `MemoryViewComponent.processData()` — verify that the `BigInt` conversion and hex padding correctly handle large address boundaries.
