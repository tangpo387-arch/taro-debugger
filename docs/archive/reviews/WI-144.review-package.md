## 3. Edge Cases & Design Decisions

- **Defensive Check for Mock Environments**: Mock `DapThreadSession` objects in unit tests lack detailed `.status` properties. Evaluating `thread.status !== 'running'` (which is `true` for suspended and undefined mock states) ensures backward compatibility without regressions.
- **Active Thread Stepping Guards**: Keyboard actions and shortcuts are prevented from sending `stepIn`/`next` to GDB if the active thread is currently running.
- **Pause Routing**: Allowed pausing if the global session is running OR the focused thread itself is running asynchronously, forwarding the specific thread ID to the DA.
