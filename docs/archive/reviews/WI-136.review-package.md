## 3. Edge Cases & Design Decisions

- **Uniform Blob Framing**: Initially the transport handled plain JSON strings for setup channel messages. Refactored to use `sendFramed()` on the server instead, keeping a single Blob code path in the transport. This prevents branching and keeps the wire format consistent for all channels. 🔍 Inspect: `server.ts` `sendFramed()` vs `sendToClientAsBlob()` — both do Content-Length framing; `sendFramed` accepts a socket parameter for direct setup sends, while `sendToClientAsBlob` uses `this.clientSocket`.
- **Log path isolation**: Default changed from `process.cwd()` (workspace root pollution) to `os.tmpdir()/taro-session-logs-<PID>/`. The PID namespace prevents log interleaving between concurrent daemon instances.
- **Evaluate timeout on fatal teardown**: `DebugConsoleComponent.ngOnDestroy()` now cancels in-flight `evaluate()` requests. The `destroyed` flag prevents `cdr.detectChanges()` from being called on a destroyed view. `cancelRequest()` is already safe to call after teardown (guarded by state check internally).
- **Reconnect-Retry Only policy**: On `ERROR`, server closes the socket immediately (Fail-Fast). Client must establish a fresh connection — no socket reuse. This matches the spec and prevents stale state.
