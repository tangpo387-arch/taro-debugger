## 3. Edge Cases & Design Decisions

- **TestSocket Helper**: Created a robust, race-free `TestSocket` and `connectTestSocket` helper inside `server.spec.ts`. This buffers all incoming socket messages from the moment of connection establishment, eliminating any race conditions where protocol payloads (such as `'chat-history'` or `'session-ready'`) arrive before the test registers its event listener.
- **Connection Error Boundaries**: Added try/catch and error-listener gate tracking to the WebSocket client connection tests. If a socket errors out (e.g. abnormal close `1006` or connection refused), the promise is cleanly rejected, letting Vitest capture the failure inside the active test lifecycle instead of throwing uncaught process-level exceptions that could crash the runner.
- **Deferred Cleanup**: Utilized a `tempDirs` queue and handled directory sweeps inside the `afterAll` hook after a short 200ms grace period. This guarantees that background file stream writes/closes complete cleanly on disk before their parent directories are deleted, eliminating `ENOENT` crashes.
