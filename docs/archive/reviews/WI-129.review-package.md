## 3. Edge Cases & Design Decisions

- `isOutOfRange()` modification: Simplifies dimming logic to ONLY target zero-padding or unmapped memory blocks. This prevents adjacent instructions of different functions from being dimmed, which solves readability issues when scrolling arbitrary memory.
- `rowHeight() * 2` height calculation: Using `[style.height.px]="instruction.isFunctionStart ? rowHeight() * 2 : rowHeight()"` on the row itself dynamically adjusts CDK virtual scroll viewport heights so that the viewport offset and scroll metrics are preserved without breaking alignments.
- Regex validation: Safety regex `^0x[0-9a-fA-F]+$` checks the address format before calling GDB evaluate commands to avoid invalid or unsafe commands to the DAP adapter.
- GDB fallback evaluation: Only triggered when a fetched batch starts without a symbol (`!firstInst.symbol`), and we check if GDB evaluate response does not indicate "no symbol" before assigning. Uses `info symbol` directly without `-exec` prefix.
- Scroll Offset-Matching Guard: Added `expectedScrollOffset` matching check to `onViewportScroll`. Programmatic scroll alignment suppresses browser scroll events carrying stale offsets. The guard is cleared instantly when the current offset matches the target offset (or clamped boundaries), preventing redundant or overlapping fetches during rapid scrolling.
- Unified Distance-Preserving Alignment: `updateInstructions()` measures the pixel distance from the viewport top to the anchor address *before* replacing the instruction list, and restores that exact distance *after* CDK layout. This unified approach covers both forward and backward page transitions, eliminating viewport jumps that triggered infinite scroll-fetch loops.
