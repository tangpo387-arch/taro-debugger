## 3. Edge Cases & Design Decisions

- **Recursive Pending**: In `DapSessionService.setBreakpoints`, the pending update is triggered via a recursive call in the `finally` block. This ensures the queue is processed regardless of success/error of the in-flight request.
- **Group Debounce**: Used RxJS `groupBy` in `EditorComponent` to ensure that clicking in File A doesn't accidentally debounce an intended immediate click in File B.
- **Return Value**: When a request is queued as pending, `setBreakpoints` returns `[]`. The UI (DebuggerComponent) handles the eventual verification state via the recursive call or DAP `breakpoint` events.
