## 3. Edge Cases & Design Decisions

- **UI Consistency**: Refactored the Memory View to mirror the Disassembly View's layout (sticky header + floating action buttons). This provides a more professional "Flush" IDE aesthetic.
- **Shared Components**: Unified the "Jump to Address" logic into a single component in `ui-shared`, reducing code duplication across `ui-assembly` and `ui-inspection`.
- **Context-Aware Wording**: The shared dialog supports custom titles and descriptions via `MAT_DIALOG_DATA`, allowing "Symbols" for Assembly and "References" for Memory.
- 🔍 **Inspect**: Verify the `floating-actions` positioning in `memory-view.component.scss` — ensured it doesn't overlap scrollbars.
