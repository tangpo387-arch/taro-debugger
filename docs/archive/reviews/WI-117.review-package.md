## 3. Edge Cases & Design Decisions

- **Decision**: Eliminated the global `Map<bigint, instruction>` and `sortedAddresses: bigint[]`. The cache now consists solely of `CachedRange[]`, where each range contains its own sorted `instructions[]` array.
- **Decision**: Shifted complexity from $O(N \log N)$ (re-sorting entire cache) to $O(K + M)$ (merging local ranges). Binary search is now performed per-range instead of globally.
- **Edge Case**: **x86 variable-length overlap**. The merge logic now applies a strict ascending address filter during both `neg/pos` concatenation and range merging to handle adapters returning misaligned instructions after a jump.
- 🔍 **Inspect**: The `mergeBatchIntoRanges` method in `dap-assembly-cache.service.ts` (L450-485). Verify that the `splice` and range-collapsing logic correctly maintains the sorted order of `cachedRanges`.
- 🔍 **Inspect**: `getFromCache` (L407-420) binary search. Confirm it correctly handles the `instructionOffset`.
