## 3. Edge Cases & Design Decisions

- **Private Access in Tests**: Existing unit tests for `DapSessionService` still need to verify low-level protocol behavior (like sequence ID increments). These were updated to use `(service as any).sendRequest` to bypass TypeScript visibility while maintaining protocol integrity for the rest of the app.
- **Getter Removal**: The synchronous `executionState` getter was removed to prevent "reactive-bypass" where components read state once and miss updates. `DebugControlGroupComponent` was refactored to use `firstValueFrom(this.dapSession.executionState$)` in its imperative click handler, ensuring it always sees the latest state.
- **Stale State Fix**: `DapVariablesService.spec.ts` was found to be using a deprecated `'terminated'` state (which was merged into `'idle'` in a previous refactoring). This was fixed to restore build stability.
- **Layout Test Removal**: `DebuggerComponent.spec.ts` contained tests for layout properties that were removed during the `TaroPanelGroupComponent` refactoring. These were removed as they are no longer relevant to `DebuggerComponent` and were blocking the test runner.
