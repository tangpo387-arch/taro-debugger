## 3. Edge Cases & Design Decisions

- **Per-Thread Stop Reasons**: Addressed QCR S1 by tracking stop reasons in a `Map<number, string>` within `DapSessionService`. This ensures multi-threaded sessions correctly display specific reasons (e.g., "exception" vs "breakpoint") per thread.
- **SSOT Enforcement**: All control methods are event-driven. The UI state transitions **only** when the DAP event stream confirms the change.
- **Resource Management**: Implemented `finally` blocks in `fetchFrames` to ensure loading states are always cleaned up, addressing QCR D1.
- **Cleanup**: Legacy components removed; CSS optimized by merging duplicate rules (QCR S3).
