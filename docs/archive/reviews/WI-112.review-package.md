## 3. Edge Cases & Design Decisions

- **Jump Context**: The service now forces a fetch when `instructionOffset < 0` (jump) even if the address is already in the UI list. This ensures we always re-center with a full +/- 1000 buffer upon an explicit PC change.
- **Signal Integration**: Used `toSignal` in `AssemblyViewComponent` for clean, reactive binding to the service's PC state.
- **Split-Request Anchoring & GDB Workaround**: Implemented a concurrent fetch strategy in `dap-assembly-cache.service.ts` to prevent DAP from swallowing the PC. Included a fallback that uses byte-math to dynamically compute a preceding `memoryReference` when GDB incorrectly returns empty lists for negative offsets.
- **Mat-Tab Visibility Toggle**: Handled an edge case where switching to the Source View causes the Assembly View's viewport size to collapse to 0. A `ResizeObserver` detects when it becomes visible again (`becameVisible`) and triggers an instant `revealPC(true)` to re-center the PC perfectly without sliding animations.
- 🔍 **Inspect**: `dap-assembly-cache.service.ts` (L110–150). Verify the split-request deduplication and GDB fallback logic correctly isolates the PC while safely managing byte-math guessing.
