## 3. Edge Cases & Design Decisions

- 🔍 Inspect: `projects/ui-shared/src/lib/panel-group/panel-group.component.ts` L94-114. This handles the math for dynamic resizing clamped by both panels' `minExpandedHeight`, including the lazy DOM read that prevents flex proportion skewing.
- Decision: Relied on native Flexbox behavior (`flex-grow: 1`) to implicitly redistribute space when panels are collapsed, removing the need for manual height redistribution tracking in code.
- Decision: Applied explicit CSS `min-height` dynamically in `applyPanelStyle` to strictly override `flex-shrink` and prevent the "disappearing panel" bug when the viewport is too small to mathematically satisfy the total `flex-basis`.
- Decision: Replaced backing TS properties (`leftFilesExpanded`, etc.) with local template reference variables (`#filesPanel`) using optional chaining `?.` to eliminate component boilerplate while preventing Angular template initialization crashes.
