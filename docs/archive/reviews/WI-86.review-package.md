## 3. Edge Cases & Design Decisions

- **Decision**: Separated `onRun()` and `onRestart()` in `DebuggerComponent`. `onRun()` always calls `startSession()`, while `onRestart()` is now strictly for active sessions.
- **Decision**: The "Restart" button is explicitly disabled in `terminated` and `idle` states. This enforces a clean restart (new session) using the "Play" (Run) button.
- **Critical Fix**: Excluded the `terminated` state from the "unexpected disconnect" error transition. This prevents the UI from flashing an error state when the DAP server closes the connection after a natural debuggee exit.
- **Guard logic**: `DapSessionService.restart()` now incorporates a safety guard that returns early if called from `idle`, `starting`, or `terminated` states.
