## 3. Edge Cases & Design Decisions

- **Thread-Persistent Cache**: Based on user feedback and memory space analysis, the cache is no longer cleared on thread switches. Since threads share the same `.text` segments, this provides zero-latency navigation when switching between threads.
- **Module Invalidation**: Added explicit cache clearing on `module` events. This ensures that if the memory map changes (e.g., dynamic loading), the cache is flushed to prevent stale instruction pointers.
- **Unified Cache Retrieval**: Refactored specialized cache methods into a single, partial-hit-friendly `getFromCache` method. This reduces architectural complexity and minimizes binary search overhead.
- **Explicit Context Requests**: Removed "magic" window expansion from the service layer. The `DebuggerComponent` now explicitly requests the desired 200-instruction window (`offset: -100`), making the protocol flow predictable.
- **Fast-Path Stepping**: Implemented a bypass for DAP disassemble requests if the target IP is already present in the current UI stream. Combined with `BigInt` address normalization, this prevents jarring view resets.
- **Reactive Safety**: Replaced manual `isDestroyed` logic with Angular 16+ `DestroyRef` and `takeUntilDestroyed`. Added explicit `clearTimeout` management for centering animations to prevent runtime errors.
- **Return to PC Button**: Implemented a floating "Reveal current execution point" button that resets the scroll-tracking state and performs a smooth centering animation to the active IP.
