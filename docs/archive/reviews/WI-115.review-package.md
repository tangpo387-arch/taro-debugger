## 3. Edge Cases & Design Decisions

- **Defensive Check**: In `GlobalFatalErrorHandler`, I check both `instanceof` and `.name === '...'`. This ensures that even if the prototype is lost during serialization (IPC) or if libraries are bundled multiple times, the fatal error is still correctly identified.
- **Zone.js / RxJS Wrapping**: The handler explicitly unwraps `error.rejection` and `error.originalError` to handle common wrapper objects used by Angular and RxJS.
- **NgZone.run**: The dialog and navigation are triggered within `zone.run()` because global error handlers may sometimes be called from outside the Angular zone (e.g., from an asynchronous background task).
- **Fatal Error UX**: The RETRY button is suppressed for fatal errors as they represent unrecoverable states that require returning to the setup page.
