## 3. Edge Cases & Design Decisions

- **Decision**: The Jump FAB is rendered first in the DOM to ensure it appears at the top of the flex-column `.floating-actions` as requested.
- **Decision**: `JumpToAddressDialogComponent` is standalone and does NOT inject `DapAssemblyService`. It returns the address string to the caller (`AssemblyViewComponent`) to maintain a clean UI/Service boundary.
- **Decision**: Scrollbar removal targets `-webkit-scrollbar`, `scrollbar-width` (Firefox), and `-ms-overflow-style` (IE/Edge) for cross-browser consistency.
- **Fix**: Resolved Jump-to-Address dialog layout issues by removing `min-width` from the form field, setting `overflow: visible`, and adding `padding-top: 16px` to prevent label truncation.
- **Rework (F1)**: Added `takeUntilDestroyed(this.destroyRef)` to the `afterClosed()` subscription in `openJumpToAddressDialog()` to comply with project memory safety standards.
- **Rework (F2)**: Fixed viewport snap-back issue by updating `lastScrolledIP` to the current PC during manual jumps. This "acknowledges" the current execution point and prevents automatic re-centering during infinite scroll data expansion.
- **🔍 Inspect**: `AssemblyViewComponent.scrollToAddress` — verify that updating `lastScrolledIP` to the current PC (L245) correctly prevents expansion-jumps while still allowing manual address jumps.
