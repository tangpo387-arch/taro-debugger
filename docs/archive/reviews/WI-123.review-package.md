## 3. Edge Cases & Design Decisions

- **Protocol Compliance**: Used explicit `setFunctionBreakpoints` instead of adapter-specific launch arguments to ensure universal compatibility (GDB/LLDB).
- **ID Isolation**: System-managed breakpoints are tracked in a private `systemBreakpointIds` set. This prevents them from appearing in the user's "Breakpoints" panel and avoids ID collisions with user-set source breakpoints. The 'breakpoint' event handler also filters out these IDs to ensure symbolic-to-source resolution events from the adapter do not inadvertently add system breakpoints to the user's view. IDs are registered immediately upon creation, even if initially unverified, to ensure they are recognized if the adapter verifies them later.
- **Stop Reason Override**: When the debugger hits the injected `main` breakpoint, the UI specifically reports "Paused at entry (main)" instead of a generic "Breakpoint hit".
- **Future-Proofing**: The implementation is designed to merge with future persistent user-defined function breakpoints by appending the system-managed `main` symbol to the declarative list.
