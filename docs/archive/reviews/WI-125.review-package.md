## 3. Design Decisions

- **Click vs Hover**: Replaced a CDK hover-overlay approach that suffered from irreconcilable virtual-scroll coordinate-space misalignment. A click-based button provides a stable, small DOM anchor (`cdkOverlayOrigin` on the button element) that is immune to parent CSS transforms and virtual scroll recycling offsets.
- **Overlay Positioning**: Anchored with `originX: 'end'` so the panel opens flush with the right edge of the action buttons, with a fallback to above when the panel bottom would clip the viewport.
- **Scope Filter**: Uses `/^(registers?)$/i` to handle both `Registers` (LLDB) and `register` (GDB) scope naming conventions.
- **Sticky Row Actions**: The `.row-actions` container uses `position: sticky; right: 4px` to keep action buttons visible at all times, even when variable names force horizontal scrolling.
