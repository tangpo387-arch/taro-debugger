## 3. Edge Cases & Design Decisions

- **Two-way binding via `[(expanded)]`**: `InspectionPanelComponent` uses `@Input() expanded` + `@Output() expandedChange` — the standard Angular banana-in-a-box pattern. The parent (`DebuggerComponent`) owns the expand state, enabling cross-panel resize logic.

- **Flex-basis approach for resize**: Instead of absolute pixel heights, panels use `[style.flex]` set to `"1 1 <height>px"` when expanded and `"0 0 32px"` when collapsed. This delegates overflow protection entirely to the sidenav's flex container. 🔍 **Inspect**: Verify that `min-height: 0` on `:host` in `inspection-panel.component.scss` is sufficient to prevent flex overflow when all panels are expanded simultaneously.

- **`@ViewChild` for sidenav bounding rect (ADV-1 fix)**: `onLeftPanelResizeDrag` and `onRightPanelResizeDrag` now resolve the sidenav `HTMLElement` via `@ViewChild('leftSidenav'/'rightSidenav', { read: ElementRef })` using template ref vars `#leftSidenav` / `#rightSidenav`. This replaces the original `document.querySelector` approach, eliminating CSS class name coupling.

- **`[Test]` criterion is layout-only**: The acceptance criterion tests that both sidenavs render `app-inspection-panel` children with 1px dividers. No DAP data wiring is tested (deferred to WI-70/71). No new `.spec.ts` file was added for this WI because the new components are pure structural placeholders with no logic.

- **`NgZone.runOutsideAngular`** is used in `InspectionPanelComponent.onResizeHandleMousedown` for the mousemove listener. Zone re-entry happens only when emitting the drag event to the parent, minimizing change detection overhead during rapid drag.
