## 3. Edge Cases & Design Decisions

- **Direct Throw during setup command handling**: Instead of silently creating directories, `open-session` now expects that the folder exists. Throwing an error within the try/catch in `handleSetupMessage` forces the socket into `ERROR` state and broadcasts a framed `session-failed` event cleanly.
- **Frontend Interception**: When `session-failed` is received, `DapSessionService` throws an exception that `DebuggerComponent` handles by displaying the MatDialog `ErrorDialog` and navigating back to `/setup`. This prevents initialization of subsequent DAP frames when the directory does not exist.
