## 3. Edge Cases & Design Decisions

- **Reactive SSOT**: Moved UI cleanup from event handlers to the central state subscription. This ensures that *all* transitions to `idle` (including manual disconnects or transport crashes) trigger the same cleanup flow.
- **Empty State Consistency**: Used `<taro-empty-state>` in the Editor tab to match the UX pattern used in the Call Stack and Variables panels.
- 🔍 **Inspect**: `DebuggerComponent.ngOnInit` (L204-222) — verify that the `idle` state correctly clears both execution state and editor state without side effects on other components.
