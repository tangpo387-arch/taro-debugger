## 3. Edge Cases & Design Decisions

- **Configuration Guard Bypass**: Under `'open'` mode, the executable path is empty on view boot because it is retrieved from the daemon handshake dynamically. We modified the `DebuggerComponent.ngOnInit()` guard to ignore empty `executablePath` when `setupMode === 'open'`.
- **Dynamic Combined Validator**: The `executablePath` input in the setup view is now required *only* when `setupMode === 'new'` AND `launchMode === 'launch'`. All other modes render it optional or hidden, preventing validation locks.
- **Fail-Fast Handshake Cleanup**: A `try-catch` wrapper around the setup handshake promise ensures that any socket terminations or timeout conditions cleanly execute `closeTransport()` and reset the execution state back to `disconnected` to prevent stale transport leaks.
- **Enhanced Open UX**: Following critical UX review, selecting 'Open Existing' now automatically hides both 'Launch Mode' and 'Stop at main entry point' parameters, as these options are loaded dynamically from the target session folder. The connection button dynamically shifts its label to 'Open & Debug' with a corresponding 'folder_open' icon to match this optimized flow.
