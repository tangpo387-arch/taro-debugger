## 3. Edge Cases & Design Decisions

- **Direct Thread Objects**: Subclassing was completely bypassed. `DapThreadSession` was placed directly as the property-holder in `DapSessionService`. All consumers interact cleanly via thread objects.
- **Debounce Optimization**: A temporal debounce buffer of 50ms is used on the internal GDB/DAP thread event listener before triggering a GDB thread refresh. High-frequency started/exited events are automatically grouped into a single GDB request, preventing N+1 thread lists.
- **Coalescing Execution**: When parallel `stackTrace` calls are initiated, they reuse a single shared Promise (`stackTracePromise`) on the thread object, preventing redundant network round-trips.
- **🔍 Inspect**: The `getOrCreateThreadObject` helper in `dap-session.service.ts` — verify that it preserves existing thread cache instances cleanly and clears them completely when the execution state machine transitions to `running` or `idle`.
