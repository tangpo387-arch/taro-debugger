## 3. Edge Cases & Design Decisions

> Explain any non-obvious decisions. Flag areas requiring deeper QCR inspection with 🔍.

- **Decision**: Used `DAP_SESSION_PROVIDERS` to bundle all sub-managers. Angular's DI system created a "split-brain" bug when the `DebuggerComponent` didn't explicitly provide `DapSessionLifecycle`, falling back to the root module. Bundling them in a single array avoids exposing internal managers to the public API while guaranteeing isolated session instances per UI view.
- **Decision**: Fixed `gdbProcess` tear-down race condition in the backend by nullifying the reference synchronously before the asynchronous teardown begins to prevent late `terminated` events triggering idle state loops.
- 🔍 **Inspect**: The Dependency Injection regression test in `projects/taro-debugger-frontend/src/app/debugger.component.spec.ts` — verify it properly tests `DapThreadManager` reference equality without `provideDapCore()` falling back to root.
