## 3. Edge Cases & Design Decisions

- **Decision**: Made `reset()` private to prevent external components from bypassing proper lifecycle methods.
- **Recovery Strategy**: Updated `stop()` and `disconnect()` to allow transitioning from `error` to `idle`. `stop()` now calls `reset()` directly when in `error` state, while `disconnect()` skips the DAP request but performs the local reset.
- 🔍 **Inspect**: Ensure that skipping the DAP `disconnect` request when in `error` state is the correct behavior (intended to avoid sending requests over potentially dead transports).
