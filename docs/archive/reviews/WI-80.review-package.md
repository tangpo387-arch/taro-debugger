## 3. Edge Cases & Design Decisions

* **`centered` input defaults to `true`**: All 5 panels use the default centered layout. The `false` path exists for future use (e.g., inline empty states inside list items), and is covered by a dedicated unit test.
* **`icon` and `description` are optional**: Both use Angular's `@if` control flow to conditionally render their elements, ensuring no empty DOM nodes are created when inputs are absent.
* **FileExplorer uses `unsupported` state (not data-empty state)**: The File Explorer's empty state triggers when the DAP adapter lacks `loadedSources` support — a distinct semantic from "no data yet." The same component is used here for visual consistency, but the icon (`info`) and description text differentiate the UX intent.
* **Threads component uses inline template**: `ThreadsComponent` uses an inline Angular template (not a separate `.html` file). The `*ngIf` structural directive was retained to match the existing pattern in this component. This is flagged for inspection.
* 🔍 **Inspect**: `threads.component.ts` uses inline template + styles, which differs from the code-style guide. The `*ngIf` pattern was retained (not migrated to `@if`) because the template is still using `*ngFor` for the thread list. Verify that the `taro-empty-state` renders correctly within the existing `*ngIf` on `ng-container`.
