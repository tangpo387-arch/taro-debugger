## 3. Edge Cases & Design Decisions

> Explain any non-obvious decisions. Flag areas requiring deeper QCR inspection with 🔍.

- Decision: For `evaluateInFlight$`, I utilized a BehaviorSubject approach instead of maintaining an unmanaged boolean flag, correctly allowing template usage.
- Decision: Overrode the `sendRequest` DAP timeout within `evaluate()` to `35000` to prevent the standard 5-second `sendRequest` timeout from throwing before `evaluate` can enforce its R-CS2 30-second timeout.
- 🔍 Inspect: Check that mapping between `DapSessionService` traffic subscription logic matches `seq` fetching for correct cancellation behavior under concurrency.
