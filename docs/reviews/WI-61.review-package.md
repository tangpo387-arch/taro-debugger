## 3. Edge Cases & Design Decisions

- **Decoupling**: `DapSessionService` no longer manages the file tree lifecycle. The UI layer (`DapFileTreeService`) owns cache invalidation by subscribing to `onEvent()` and `connectionStatus$`.
- **DI Scope — R_SM5.2 (relaunch bug)**: `DapFileTreeService` must be component-scoped (declared in `DebuggerComponent.providers`) because it depends on the component-scoped `DapSessionService`. Using `providedIn: 'root'` caused it to receive the uninitialised root instance — triggering "Transport not initialized" errors on every file read. Fixed by removing `providedIn: 'root'` and adding `DapFileTreeService` to `DebuggerComponent.providers`.
- **Transport Subject Lifecycle (relaunch bug)**: `WebSocketTransportService` and `IpcTransportService` are root-level singletons. After `disconnect()`, their `messageSubject` was in a terminal (completed/error) state. On the second session,  `DapSessionService` subscribed to the already-completed stream and immediately received a close signal, causing the `initialize` request to time out. Fixed by recreating `messageSubject` on every `connect()` and `disconnect()`.
- 🔍 **Inspect**: `WebSocketTransportService.disconnect()` — verify handlers are cleared (`onopen`, `onmessage`, `onerror`, `onclose` set to `null`) before `socket.close()`, preventing double-fire on the new session's event loop.
- 🔍 **Inspect**: `DapFileTreeService.destroy()` — JSDoc still references `DapSessionService.reset()` as the caller. This is stale; the caller is now `ngOnDestroy` on the component.
