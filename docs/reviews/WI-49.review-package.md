## 3. Edge Cases & Design Decisions

- **Restore Timing**: Restoring the view state is performed within the `updateQueue$` subscriber (debounced 50ms). This ensures that the Monaco model has likely finished updating its content before the position is restored.
- **Conflict Resolution**: If `activeLine` is provided (e.g., hitting a breakpoint after manual navigation), `scrollToLine` is called *after* `restoreViewState`, ensuring the execution context highlight takes precedence over the saved scroll position.
- **Identity Guard**: Added `lastRestoredFilename` to prevent redundant `restoreViewState` calls when `activeLine` or `code` updates occur within the same file.
