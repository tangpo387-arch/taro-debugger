## 3. Edge Cases & Design Decisions

- **Silent drop strategy**: Per `command-serialization.md §2 (R-CS1)`, a second control command dispatched while one is in-flight is silently dropped. It returns a dummy `DapResponse { seq: 0, success: true }` to prevent unhandled rejection noise in callers.
- **Async/Event decoupling**: `commandInFlightSubject.next(false)` is fired in **both** the `finally` block of the guarded methods AND on receipt of `stopped` / `continued` / `terminated` / `exited` DAP events, and on transport error/reset. This ensures buttons unlock even if the DAP adapter sends an event out-of-order before resolving the response.
- 🔍 Inspect: `handleTransportEvent` — verify each state case correctly calls `commandInFlightSubject.next(false)`.
- 🔍 Inspect: `dap-session.service.spec.ts` L188 — verify no `subscribe()` without teardown remains.
