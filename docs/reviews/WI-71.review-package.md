## 3. Edge Cases & Design Decisions

- **Decision: Preserve Enabled State**: When a file is re-synced (e.g. adding a new breakpoint), the `setBreakpoints` logic in `DapSessionService` merges the new verification results with the existing local `enabled` states to prevent user preference loss.
- **Decision: Partial Sync**: Only `enabled: true` breakpoints are sent to the DAP adapter. Disabled breakpoints are kept in the SSOT but removed from the adapter session to "disable" them.
- **🔍 Inspect: `DapSessionService.setBreakpoints` L405–444**: Verify the merge logic correctly handles relocated breakpoints while preserving the `enabled` flag.
- **🔍 Inspect: `EditorComponent` L345–360**: Verify the decoration class precedence (Disabled > Unverified > Verified).
